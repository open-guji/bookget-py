# 识典古籍

北京大学与字节跳动（抖音集团）联合建设的免费古籍数字化阅读平台，依托"北京大学-字节跳动数字人文开放实验室"，提供古籍全文检索、底本影像对照阅读、OCR 识别文字等功能。2022 年 10 月上线。

- 网站链接：https://www.shidianguji.com/
- 分享协议：未公开具体数据使用协议。平台特点为"不限连接数、不需要登录、不限制复制"
- 约 19,000+ 部古籍（2025 年数据），目标 2025 年底完成 25,000 种
- 资源类型：经部、史部、子部、集部、道教部、佛教部（六部分类），以《四部丛刊》为基础扩展
- 不支持 IIIF 协议
- bookget Go 版本不支持此站点

## 搜索与浏览

- 书库页面：https://www.shidianguji.com/library
- 在搜索框输入关键词即可搜索，支持按书名、作者、全文段落检索
- 可按分类（经/史/子/集/道/佛）和朝代筛选
- **不需要注册登录**即可阅读和检索
- 支持多语言界面（中文简体/繁体/英文），URL 前缀 `/zh/`、`/en/`

## 每本书能看到什么

| 字段 | 说明 | 示例 |
|------|------|------|
| 书名 | 书名（含繁简） | 周易 |
| 著者 | 作者/注者/编者，标注朝代 | 佚名 著、韩康伯 著、邢璹 著 |
| 分类 | 六部分类法 | 经部 |
| 底本影像 | 原书扫描图片 | 可在线逐页翻阅 |
| OCR 文字 | 自动识别的全文文字 | 准确率 96-97% |
| 自动标点 | AI 自动断句标点 | 有 |
| 命名实体 | 人名、地名、时间标注 | 有（可点击跳转） |

特色功能：
- 底本影像与 OCR 文字对照阅读
- AI 智能助手（大模型，2024 年上线），可进行古文问答
- 全文段落级检索
- 地图可视化（地名实体关联）
- 音频朗读（部分书籍）

# 开发者文档

## URL 结构

### 书籍阅读页
```
https://www.shidianguji.com/{lang}/book/{book_id}
https://www.shidianguji.com/{lang}/book/{book_id}?page_from=home_page
```
- `lang`：语言代码，`zh`/`en`/`ens`（英文简化版）
- `book_id`：书籍 ID，如 `SBCK001`（四部丛刊编号）、`7282969127763836969`（其他编号）
- `page_from`：来源跟踪参数（可选）

### 章节阅读页
```
https://www.shidianguji.com/{lang}/chapter/{chapter_id}
```

### 搜索页
```
https://www.shidianguji.com/search/{query}
```

## 技术架构

**前端**：SPA 单页应用（基于字节跳动 SSR 框架），HTML 源码不包含实际内容数据。

- 静态资源 CDN：`lf-welfare.amemv.com/obj/douyin-welfare-image/guji/shidian/`
- UI 库：Semi Design（字节跳动开源组件库）
- HTTP 客户端：Axios
- 安全 SDK：字节跳动 BDMS + Slardar 性能监控 + SecSDK 反爬

**SSR 数据**：页面中有 `window._ROUTER_DATA` 对象，包含 i18n 翻译数据（书名、作者的本地化文本），但**不包含**书籍详情、页面列表等核心数据。核心数据通过客户端 API 请求加载。

## API 概览

### 基础信息

- API 前缀：`/api/ancientlib/read/`（公开读取 API）
- 内部 API 前缀：`/internal-api/ancientlib/read/`（SSR 服务端调用）
- 管理 API 前缀：`/api/ancientlib/mgt/`（编辑/管理后台）
- 请求方式：GET 或 POST（参见各接口说明）
- 数据格式：JSON
- 认证：`withCredentials: true`，需要字节跳动设备指纹 SDK 生成的 cookie/签名

### ⚠️ 反爬机制

所有 API 调用均返回 `{"errorCode": 40001, "errorMsg": ""}` 除非携带有效的设备指纹信息。

网站使用字节跳动安全 SDK（SecSDK + BDMS），通过 JavaScript 在客户端生成设备指纹和签名 cookie。**无法通过简单的 curl/requests 直接调用 API**。

已知的错误码：
- `40001`：认证失败（缺少设备指纹）
- `40032`：超出限制
- `40033`：书架已满
- `40034`：ptoken 为空
- `40035`：ptoken 无效

- `isWebpSupported`：是否支持 WebP
- `thumbResizeWidth` / `thumbQvalue`：缩略图参数
- `pagination`：分页信息
- `imageResizeWidth` / `imageQvalue`：图片参数

#### `searchBookV1`（搜索书籍）
```
POST /api/ancientlib/read/search/book/v1
```
参数：
- `queryText`：搜索关键词

- `traditionalCategory`：传统分类
- `dynastyCategory`：朝代分类
- `pageNum` / `pageSize`：分页

#### `batchGetBook`（批量获取书籍）

## 认证机制（已验证）

**API 受字节跳动 SecSDK 保护**，无法通过简单的 curl/requests 调用（返回 `errorCode: 40001`）。

通过 Playwright 浏览器自动化**已验证可以正常访问所有 API**（`errorCode: 0`）。

### 签名参数

SecSDK 自动在每个 API 请求 URL 中附加两个签名参数：
- `verifyFp`：设备验证指纹（如 `verify_mmmiob7b_4xw8jeYW_O6Kr_4rhc_9tnR_VqzuH1iLle9Q`）
- `a_bogus`：请求签名（每次请求不同，由 JS 实时计算）

另外还附加 `fp` 参数（值同 `verifyFp`）。

### 关键 Cookie

| Cookie | 说明 |
|--------|------|
| `ttwid` | 字节跳动设备标识（JS 自动生成） |
| `s_v_web_id` | 验证指纹 ID（格式 `verify_xxx`） |
| `ptoken` | 页面令牌，用于图片 URL 解密 |
| `x-web-secsdk-uid` | SecSDK 设备 UUID |
| `gfkadpd` | 应用/页面 ID（固定值 `361265,34425`） |

### 访问方式

推荐使用 Playwright 浏览器自动化：加载书籍页面后 JS SDK 自动完成认证，拦截 API 响应即可获取数据。

### 图片下载（已验证可行）

图片托管在字节跳动 CDN（`byteimg.com`），**图片本身可直接下载，无需认证**。

#### 图片 URL 格式（已验证）
```
https://p{N}-ancientlib-sign.byteimg.com/tos-cn-i-ceu0di7tn2/{uri}~tplv-ceu0di7tn2-image.image?lk3s=e5c95217&x-expires={timestamp}&x-signature={signature}
```

- `{N}`：CDN 节点编号（1-9，随机）
- `tos-cn-i-ceu0di7tn2`：字节 TOS 存储前缀（固定）
- `{uri}`：从 `bookPagesV3` API 响应的 `pages[].uri` 字段获取，如 `read/SBCK001/4/page/1jbe7jgxjjw2t/1iwgll61v6b04-SBCK001_00001_00001A.webp`
- `~tplv-ceu0di7tn2-image.image`：图片处理后缀（固定）
- `lk3s=e5c95217`：存储桶标识（固定）
- `x-expires=1778112000`：URL 过期时间戳（约数月有效期）
- `x-signature`：URL 签名（从拦截的浏览器图片请求 URL 中提取）

注意：`pages[].picUrl` 是加密字符串（`urlEncryptType: 1`），不能直接使用。真实 URL 需通过拦截浏览器图片请求获取。

#### 下载流程（Playwright）
1. 用 Playwright 加载书籍页面，等待 JS SDK 完成认证
2. 拦截浏览器发出的图片请求（`*ancientlib-sign.byteimg.com*`），提取完整签名 URL
3. 直接 HTTP GET 签名 URL 下载图片（**无需额外认证头**）

#### 实测结果（SBCK001 周易，第 1 页）
- 文件大小：115,766 字节（约 113 KB）
- 分辨率：817×1287 像素
- 格式：WebP
- `x-expires=1778112000`（2026 年 4 月到期）

### 文字下载

文字内容**未加密**（`contentEncryptType: 0`），可直接从 API 响应中提取。

下载流程：
1. 通过 Playwright 加载书籍页面
2. 拦截 `bookParagraphsV2` API 响应
3. 从 `paragraphs[].content` 提取原文（JSON 格式，含 lines/sentences 层级）
4. 从 `paragraphs[].translateContent` 提取白话文翻译（如有）
5. 并发请求数受 `paragraphReqConcurrentNum`（=5）和 `paragraphReqMaxSize`（=20）限制

### bookget 支持

- bookget Go 版本：**不支持**
- 本项目适配器（`bookget/adapters/chinese/shidianguji.py`）：**已失效**
  - 原因：原 API 端点 `https://dushu.qq.com/cgi-bin/app_portal` 已 301 重定向到 `https://book.qq.com`（QQ 阅读），不再可用
  - 识典古籍早期可能使用了腾讯/阅文的后端基础设施，后迁移到字节跳动自有基础设施
  - 需要基于新的 `/api/ancientlib/read/` API 体系重写适配器，使用 Playwright 进行浏览器自动化

### 适配器重写要点

1. **认证**：必须使用 Playwright 加载页面，让 JS SDK 自动完成设备指纹生成
2. **元数据**：从 `readerBookGet` 响应的 `bookInfo` 提取
3. **图片列表**：从 `bookPagesV3` 获取，`uri` 字段包含相对路径
4. **图片下载**：拦截浏览器图片请求（`*ancientlib-sign.byteimg.com*`）获取签名 URL，然后直接 HTTP GET 下载（无需认证）
5. **文字内容**：从 `bookParagraphsV2` 获取，`content` 字段为 JSON 格式，`contentEncryptType: 0` 无需解密
6. **并发限制**：段落请求并发数 5，每次最多 20 条
